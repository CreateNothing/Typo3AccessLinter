ARIA roles and accessible names

This inspection validates ARIA roles and common accessible-name patterns and offers safe auto‑fixes. It helps keep visible labels and programmatic names in sync so assistive tech and voice control users can operate UI reliably.

Why this matters

If you’ve never heard of WCAG: think “say what you see.” When a button or link has visible words (like “Google”), assistive tools should expose the same words so people can speak or find them. If aria-label changes the words (e.g., “Going to School”), voice control and some screen reader flows won’t match what’s on screen.

The guideline behind this is WCAG 2.5.3 “Label in Name.” In plain terms: the programmatic name (e.g., aria-label) must include the visible label (ideally at the beginning).

What’s checked

  • Invalid/abstract or redundant ARIA roles.
  • Required ARIA properties for specific roles.
  • Conflicts (e.g., aria-hidden on focusable/interactive elements).
  • Accessible name best practices for links and buttons (Label‑in‑Name).
  • Unnecessary or overly long aria-label values, and instructional text that belongs in aria-describedby.
  • aria-labelledby references to missing element IDs.

Clear guidance for “Label in Name”

If the warning says “Accessible name should include the visible label,” it means your aria-label doesn’t start with the visible text. Fix by starting the label with the visible words, or remove aria-label entirely so the visible text is used.

Examples

Problem — accessible name doesn’t include the visible label:

<a href="https://example.com" aria-label="Go to Example">Example</a>

Better — start with the visible label (simple rule: say what you see first):

<a href="https://example.com" aria-label="Example — company site">Example</a>

Also good — keep aria-label short and move details to aria-describedby:

<span id="ex-desc" class="sr-only">company site</span>
<a href="https://example.com" aria-describedby="ex-desc">Example</a>

Quick‑fixes offered

  • Remove redundant role words from aria-label (e.g., “button”, “link”).
  • Remove unnecessary aria-label that duplicates visible text.
  • Shorten overly long aria-label and add aria-describedby for details.
  • Prepend visible text to aria-label to satisfy “Label in Name”.
  • (Planned) Create missing IDs for aria-labelledby references.
