Heading hierarchy

This inspection highlights skipped heading levels, multiple H1s in the same context, empty headings, and navigation headings that may crowd out the main page heading.

Why this matters

Headings convey the outline of a page. Assistive technologies use them to jump and scan. Large jumps or empty headings make navigation harder.

What’s checked

  • Multiple H1s in main content vs. sectioning contexts.
  • Skipped levels (e.g., H2 → H5) with context awareness.
  • Empty headings.
  • Navigation headings that should be lower level or expressed via aria-label on <nav>.

Quick‑fixes

  • Change heading level (e.g., H5 → H3).
  • Add placeholder text to empty headings.
  • (Planned) Normalize large jumps automatically (reduce to at most +1).
  • (Planned) Promote orphan subheadings at the start of a region.

Examples

Problem — large jump:

<h2>Section</h2>
<h5>Detail</h5>

Fix — normalize:

<h2>Section</h2>
<h3>Detail</h3>
